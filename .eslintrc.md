# ESLint Configuration Guide

## Quick Start

### Running ESLint

```bash
# Lint all TypeScript files
pnpm lint

# Auto-fix errors where possible
pnpm lint:fix

# Lint only packages
pnpm lint:packages

# Lint only scripts
pnpm lint:scripts

# Type check without emitting files
pnpm type-check
```

### VSCode Integration

1. **Install Extension**:
   - Open VSCode
   - Install "ESLint" extension (dbaeumer.vscode-eslint)
   - VSCode will prompt you on first open (already configured in `.vscode/extensions.json`)

2. **Auto-fix on Save**:
   - Already enabled in `.vscode/settings.json`
   - Files will be auto-fixed when you save (Cmd+S / Ctrl+S)

3. **Verify It Works**:
   - Open any `.ts` file
   - You should see squiggly lines under linting errors
   - Hover over errors to see ESLint messages

## Key Rules Enforced

### Google TypeScript Style Guide Compliance

✅ **Naming Conventions:**

- Classes/Interfaces/Types: `UpperCamelCase`
- Variables/Functions: `lowerCamelCase`
- Constants: `CONSTANT_CASE` or `camelCase`
- NO underscore prefixes for private fields

✅ **Type System:**

- Primitives over wrappers (`string` not `String`)
- `unknown` instead of `any`
- `interface` for objects, `type` for unions
- Array types: `string[]` for simple, `Array<T>` for complex

✅ **Imports/Exports:**

- **Named exports only** (no default exports)
- Organized import groups with newlines
- Alphabetical order within groups

✅ **Iteration:**

- `for...of` for arrays (never `for...in`)
- No `forEach` - use `for...of` or `map`/`filter`

✅ **Prohibited:**

- `namespace` keyword
- `require()` - use ES6 imports
- `@ts-ignore` without description
- `var` keyword
- Array/Function constructors

## Common Issues & Fixes

### Issue: "Unexpected any"

```typescript
// ❌ BAD
function process(data: any) { ... }

// ✅ GOOD
function process(data: unknown) { ... }
// or with specific type
function process(data: MyType) { ... }
```

### Issue: "Import should occur before"

```typescript
// ❌ BAD (wrong order)
import { something } from './local';
import fs from 'fs';

// ✅ GOOD (groups ordered)
import fs from 'fs';

import { something } from './local';
```

### Issue: "Use ReadonlyArray<T> for readonly arrays"

```typescript
// ❌ BAD
const items: readonly string[] = [];

// ✅ GOOD
const items: ReadonlyArray<string> = [];
```

### Issue: "No default export"

```typescript
// ❌ BAD
export default class MyClass { ... }

// ✅ GOOD
export class MyClass { ... }
```

## Disabling Rules (Use Sparingly!)

### Disable for a single line:

```typescript
// eslint-disable-next-line @typescript-eslint/no-explicit-any
const data: any = parseUnknownData();
```

### Disable for a block:

```typescript
/* eslint-disable @typescript-eslint/no-explicit-any */
function legacyCode() {
  const data: any = ...;
}
/* eslint-enable @typescript-eslint/no-explicit-any */
```

### Disable for entire file (avoid!):

```typescript
/* eslint-disable @typescript-eslint/no-explicit-any */
// ... file content
```

## Troubleshooting

### "Cannot find module" errors

```bash
# Reinstall dependencies
pnpm install

# Reload VSCode window
Cmd+Shift+P -> "Developer: Reload Window"
```

### ESLint not running in VSCode

1. Check output: View -> Output -> Select "ESLint" from dropdown
2. Verify extension is enabled: Extensions -> Search "ESLint" -> Should show "Enabled"
3. Check settings: `.vscode/settings.json` should have `"eslint.enable": true`

### Performance issues with monorepo

The config uses `project` references which can be slow. If linting is too slow:

1. Use targeted commands: `pnpm lint:packages` or `pnpm lint:scripts`
2. Consider disabling type-aware rules in development (not recommended)
